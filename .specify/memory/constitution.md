# PaperTrail 项目章程

## 核心原则

### 一、结构化记录

每篇论文的阅读记录必须遵循统一的结构：
- 论文元数据（标题、作者、发表会议/期刊、年份、URL/DOI）
- 核心贡献摘要（一段话）
- 关键发现与方法论笔记
- 个人见解、疑问与评价

理由：统一的结构便于长期检索和跨论文比较。

### 二、忠实呈现

笔记必须准确反映原论文的主张和贡献：
- 明确区分论文的观点与个人意见
- 对推测性或不确定的解读必须显式标注
- 对关键定义或结果应引用原文
- 绝不歪曲或夸大论文的发现

理由：阅读笔记的首要价值在于作为可靠的参考；
不准确的笔记比没有笔记更糟糕。

### 三、按主题组织

论文必须按主题目录进行组织：
- 每个主题目录包含相关的论文记录
- 当论文数量增长时，主题下可以包含子主题
- 应当标注相关论文之间的交叉引用
- 文件和目录命名必须具有描述性且保持一致

理由：按主题组织有助于知识积累，
便于纵览某一研究领域。

### 四、渐进式知识构建

每篇新的论文记录应当与已有知识建立联系：
- 记录与已读论文之间的关系
- 识别论文间的一致、矛盾或扩展关系
- 更新对更广泛研究领域的理解
- 在有益时维护轻量级的索引或目录

理由：孤立的笔记会失去价值；
相互关联的笔记能积累为对研究领域的真正理解。

## 内容标准

- 语言：笔记可以用中文或英文撰写，
  根据读者偏好选择。单篇笔记内必须保持语言一致。
- 原始 PDF 或参考副本可以与笔记一起存储，以便查阅。
- 鼓励对非母语论文进行翻译或摘要，
  但必须明确标注为翻译/摘要。
- 避免存储可在线免费获取的大型二进制文件，
  除非需要离线访问。

## 工作流程

1. **添加论文**：将 PDF（可选）放入对应主题目录，
   并创建结构化笔记文件。
2. **记录笔记**：按照原则一填写元数据、摘要、
   关键发现和个人见解。
3. **建立关联**：按照原则四，链接仓库中已有的相关论文。
4. **回顾**：随着理解的深入，定期回顾和完善笔记。

## 治理

- 本章程定义了 PaperTrail 项目的指导原则。
  所有贡献必须符合这些原则。
- 修订本章程需要更新版本号，
  并在同步影响报告注释中记录变更。
- 版本号遵循语义化版本规范：
  - MAJOR：原则移除或不兼容的重新定义
  - MINOR：新增原则或实质性扩展
  - PATCH：澄清和措辞修正
